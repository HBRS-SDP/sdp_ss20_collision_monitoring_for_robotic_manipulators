1 # Collision monitoring
2 A framework independent c++ library for monitoring the distance from any
3 manipulator to obstacles, including other manipulators in the workspace. It
4 implement concepts from these publications:
5 
6 * Real-Time Obstacle Avoidance for Manipulators and Mobile Robots,
7  Oussama Khatib, 1986.
8 * Biologically-inspired dynamical systems for movement generation:
9  automatic real-time goal adaptation and obstacle avoidance,
10  H. Hoffmann et al., 2019.
11 
12 This library was develop for the software development project course at the
13 Hochschule Bonn-Rhein-Sieg.
14 
15 ## Getting Started
16 Copy this folder to your project repository, then install the dependencies and
17 the library by following the instructions.
18 
19 
20 ### Prerequisites
21 To use this library you need the following dependencies:
22 
23 - [KDL](https://www.orocos.org/kdl/installation-manual)
24 - [Eigen 3.3.7](http://eigen.tuxfamily.org)
25 - [CMAKE V3.5](https://cmake.org/download/)
26 
27 ### Installing
28 First, in CMakeList.txt add the subdirectory and include the header files in
29 `include_directories` alongside the dependencies:
30 
31 ```CMake
32 add_subdirectory(collision_monitoring)
33 
34 include_directories(
35  ...
36  collision_monitoring/include
37  /usr/include/eigen3
38  ${Eigen_INCLUDE_DIRS}
39  ${Boost_INCLUDE_DIR}
40  ${orocos_kdl_INCLUDE_DIRS}
41  ...
42 )
43 
44 ```
45 
46 Second, for every type of manipulator you would like to monitor you must
47 implement the Arm interface. This ensure that the geometric descriptions
48 used to monitor the distances suit your manipulator. Please refer to the
49 documentation on how to do so.
50 
51 After implementing the arm you can build your project and use the monitor class.
52 If you have not done so, create a build directory. Then change directories to
53 the build directory and build using cmake:
54 
55 ```
56 mkdir build
57 cd build
58 cmake ..
59 make
60 ```
61 
62 ## Running the tests
63 After building you may run the test located in the build directory by running
64 the command:
65 
66 ```
67 ./build/test/tests
68 ```
